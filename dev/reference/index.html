Operations

insert! extends Base.insert!, so it is documented explicitly here; the remaining operations and types are listed below.

Base.insert! — Method
insert!(s, key, value; valid_from, valid_to = MAX_DATE, ts = now())

Record a new fact over [valid_from, valid_to). Returns the stored Record.

source
BiTemporalData.MAX_DATE — Constant

Sentinel for an open-ended valid_to: typemax(Date).

source
BiTemporalData.MAX_DT — Constant

Sentinel for an open tx_to: typemax(DateTime). tx_to == MAX_DT means currently believed.

source
BiTemporalData.BitemporalStore — Type
BitemporalStore{K,V}

Store of V values keyed by K. Backends implement get_records, put_record!, close_tx!, entities and inherit insert!, correct!, amend!, as_of, history, snapshot.

source
BiTemporalData.MemoryStore — Type
MemoryStore{K,V}()

In-memory reference backend. Records are kept per key in append order; id is (key, index) so close_tx! is an O(1) lookup. Not thread-safe; wrap in ThreadSafe for concurrent use.

source
BiTemporalData.Record — Type
Record{V}

Append-only bitemporal record: value over half-open [valid_from, valid_to) (valid time) and [tx_from, tx_to) (transaction time). Only tx_to may change (see close_tx!). id is backend-assigned by put_record!.

source
BiTemporalData.ThreadSafe — Type
ThreadSafe(store)

Wrap any BitemporalStore with a single store-wide ReentrantLock. Each operation locks once and runs its full primitive sequence atomically, so multi-primitive writes (correct!, amend!) cannot interleave. This is safety, not concurrency: one operation runs at a time, store-wide.

source

BiTemporalData.amend! — Method
amend!(s, key, value; effective, ts = now())

"The world changed on effective." Close the believed chapter(s) covering effective, re-append the old value over [valid_from, effective), and append value from effective on. Errors if nothing covers effective.

source
BiTemporalData.as_of — Method
as_of(s, key; valid_at = today(), tx_at = now()) -> Union{V,Nothing}

The value believed at tx_at to hold at valid_at, or nothing.

source
BiTemporalData.close_tx! — Function

close_tx!(s, id, ts) — set the record's tx_to to ts. Idempotent.

source
BiTemporalData.correct! — Method
correct!(s, key, value; valid_from, valid_to = MAX_DATE, ts = now())

"We were wrong." Close every believed record overlapping the range, then append the corrected value. History stays readable via as_of at an earlier tx_at.

BiTemporalData.snapshot — Method
snapshot(s; valid_at = nothing, tx_at = now()) -> NamedTuple of column vectors

Columnar point-in-time view of the whole store — the read boundary for bulk workloads, since freezing tx_at is reproducible and leakage-proof. With valid_at = nothing, one row per record believed at tx_at, columns entity, value, valid_from, valid_to. With a valid_at::Date, the collapsed cross-section: columns entity, value, one row per entity that has a value there. Tables.jl-compatible.
